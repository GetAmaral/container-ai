# Story 1.1: Receber e Validar Webhook da Hotmart

## Status
Draft

## Executor Assignment
executor: "@dev"
quality_gate: "@architect"

## Story
**As a** sistema backend,
**I want** receber webhooks da Hotmart, validar autenticidade e rotear por tipo de evento,
**so that** eu possa processar compras, reembolsos e chargebacks de forma segura e confiavel.

## Acceptance Criteria

1. Endpoint POST /webhook/hotmart recebe payloads da Hotmart v2.0
2. Validacao de autenticidade via hottok ou header x-hotmart-signature rejeita requests invalidos com HTTP 401
3. Idempotencia por order_id: se order_id ja foi processado, retorna HTTP 200 sem reprocessar
4. Endpoint SEMPRE retorna HTTP 200 imediatamente (processamento async)
5. Roteamento por evento:
   - PURCHASE_APPROVED -> aciona flow de ativacao (Story 1.2)
   - PURCHASE_REFUNDED -> aciona flow de desativacao (Story 1.6)
   - CHARGEBACK -> aciona flow de desativacao (Story 1.6)
   - Demais eventos -> loga e ignora
6. Payloads invalidos (JSON malformado, campos obrigatorios ausentes) retornam HTTP 400
7. Todos os webhooks recebidos sao logados (evento, order_id, timestamp, status do processamento)
8. Hotmart reenvia webhook ate 5x se nao receber 200 — sistema deve ser resiliente a duplicatas

## Tasks / Subtasks

  - [ ] Definir rota no router/controller
  - [ ] Aceitar Content-Type application/json
  - [ ] Retornar HTTP 200 imediatamente antes do processamento (AC: 4)

  - [ ] Extrair hottok do payload ou x-hotmart-signature do header
  - [ ] Comparar com token configurado em variavel de ambiente HOTMART_HOTTOK
  - [ ] Rejeitar com HTTP 401 se invalido

- [ ] Task 3: Implementar idempotencia por order_id (AC: 3, 8)

- [ ] Task 4: Implementar roteamento por tipo de evento (AC: 5)
  - [ ] Extrair campo "event" do payload
  - [ ] PURCHASE_APPROVED -> enfileirar/chamar servico de ativacao

- [ ] Task 6: Logging de webhooks (AC: 7)
  - [ ] Logar cada webhook recebido: evento, order_id, timestamp, resultado
  - [ ] Logar erros de validacao separadamente

- [ ] Task 7: Testes
  - [ ] Teste unitario: validacao de hottok (valido e invalido)

### Notas Importantes
- Hotmart reenvia webhook ate 5x se nao receber HTTP 200
- Responder 200 ANTES de processar (async) para evitar timeout e retry desnecessario
- hottok eh um token unico por conta Hotmart, configurado no painel do produtor
- Eventos ignorados: AWAITING_PAYMENT, PURCHASE_DELAYED, PURCHASE_CANCELED, PURCHASE_COMPLETE, REFUND_REQUEST, SUBSCRIPTION_CANCELLATION
