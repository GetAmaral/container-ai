# Story 1.2: Salvar User e Normalizar Telefone

## Status
Draft

## Executor Assignment
executor: "@dev"
quality_gate: "@data-engineer"

## Story
**As a** sistema backend,
**I want** salvar os dados do comprador no banco e normalizar o telefone para formato WhatsApp,
**so that** eu tenha um registro completo do user pronto para ativacao via WhatsApp ou email.

## Acceptance Criteria

1. Ao receber evento PURCHASE_APPROVED, criar ou atualizar registro do user no banco
2. Dados salvos: email, nome, primeiro_nome, ultimo_nome, telefone_normalizado, documento, tipo_documento, order_id, produto, status, data_compra
3. Normalizacao do telefone: concatenar checkout_phone_code + checkout_phone = formato "554399999999"
4. Se checkout_phone estiver vazio/null, campo telefone_normalizado fica null e flag "sem_telefone" = true
5. Se user com mesmo email ja existe no banco, atualizar dados (nao duplicar)
6. Status inicial do user: "pendente_ativacao"
7. Apos salvar, emitir evento/chamar proximo step:
   - Com telefone -> acionar envio WhatsApp (Story 1.3)
   - Sem telefone -> acionar envio email (Story 1.4)

## Dev Notes

### Normalizacao de Telefone - Exemplos
```
checkout_phone_code: "55", checkout_phone: "43999999999"  -> "554399999999"
checkout_phone_code: "55", checkout_phone: "(43) 99999-9999" -> "554399999999"
checkout_phone_code: "1",  checkout_phone: "2125551234"   -> "12125551234"
checkout_phone_code: "",   checkout_phone: ""              -> null (sem_telefone=true)
checkout_phone_code: "55", checkout_phone: null            -> null (sem_telefone=true)
```

### Status do User (ciclo de vida)
```
pendente_ativacao -> ativado -> desativado (refund/chargeback)
```

### Notas Importantes
- Email eh o identificador unico do user (vem da Hotmart)
- Telefone pode nao vir — produto pode nao exigir no checkout
- DDI internacional: nao assumir que eh sempre 55 (Brasil)
- O campo whatsapp_vinculado sera preenchido na Story 1.5 (onboarding)

### Testing
- Cobrir edge cases de normalizacao: com mascara, sem mascara, internacional, vazio
- Testar upsert (create or update) por email
