## Story
**As a** user que comprou um produto e tem telefone cadastrado,
**I want** receber uma mensagem de boas-vindas no WhatsApp automaticamente,
**so that** eu saiba que minha compra foi confirmada e minha conta ja esta ativa.

## Acceptance Criteria

1. Apos normalizacao do telefone (Story 1.2), sistema envia mensagem WhatsApp para o numero normalizado
2. Mensagem de boas-vindas contem: nome do user, nome do produto, confirmacao de que a conta esta ativa
3. Se envio for bem-sucedido, user eh marcado como "ativado" no banco
4. Se envio falhar, sistema faz retry automatico ate 3 tentativas
5. Intervalo entre retries: progressivo (ex: 30s, 2min, 5min)
6. Apos 3 falhas consecutivas, acionar fallback: envio de email (Story 1.4) e logar motivo da falha
7. Cada tentativa de envio eh logada (timestamp, status, erro se houver)
8. Nao enviar mensagem duplicada se user ja esta "ativado"

## Dev Notes

### Providers de WhatsApp Business API
Opcoes comuns:
- WhatsApp Business API oficial (Meta)
- Twilio WhatsApp
- Z-API
- Evolution API (open source)

Definir qual provider antes de implementar. A interface deve ser abstraida para trocar provider facilmente.

### Retry Strategy
```
Tentativa 1: t+0s     (imediata)
Tentativa 2: t+30s    (backoff)
Tentativa 3: t+150s   (2min30 apos inicio)
Fallback:    t+150s+  (email)
```

### Motivos Comuns de Falha WhatsApp
- Numero nao tem WhatsApp
- Numero invalido/formato errado
- User bloqueou mensagens de empresas
- Rate limit da API
- Timeout do provider

### Notas Importantes
- User do Path A (com telefone) ja fica ativado apos envio bem-sucedido
- NAO precisa de onboarding (email + codigo) — telefone ja veio da Hotmart
- Se falhar 3x, cai no mesmo email do Path B
