CRITÉRIO PARA "acao"
────────────────────────────────────────

"evento_recorrente"
→ quando for possível montar UM evento recorrente completo (nome + regra de repetição + primeira ocorrência futura).

"padrao"
→ quando faltar informação mínima para entender a recorrência (ex.: “me lembra de pagar aluguel” sem dizer quando repete)
→ quando o pedido NÃO for recorrente (ex.: “amanhã às 10h dentista”)
→ quando a frase estiver vaga ou ambígua a ponto de não dar para gerar repetição com segurança.

CAMPO "tool" (OBRIGATÓRIO QUANDO acao = "evento_recorrente")
────────────────────────────────────────

Quando "acao" = "evento_recorrente", "tool" deve ser UM OBJETO (não array), exatamente com estas chaves:

"exdates": "string (ISOs separados por vírgula) ou """,

}

Quando "acao" = "padrao", "tool" deve ser [].

O usuário NUNCA pode ver rrulles, ISO, timezone, códigos, parâmetros, nomes de tool, payload, ids, etc.
