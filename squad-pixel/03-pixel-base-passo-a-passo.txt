## DIAGNOSTICO

### Por que NAO usar o snippet padrao do Facebook?
  O snippet que o Facebook te deu para colar no <head> tem 3 problemas:
  1. Carrega ANTES do consentimento (viola LGPD)
  2. Nao funciona bem com SPA (React Router)
  3. Nao permite controle programatico de eventos

### Solucao Aplicada
  - Pixel encapsulado em modulo TypeScript (facebookPixel.ts)
  - Hook React (usePixelInit.ts) que:
    → Ouve evento de consentimento do banner
    → Checa consentimento salvo ao montar
    → Dispara PageView a cada navegacao (fix SPA)
  - Pixel so carrega apos consentimento = LGPD compliant

### Por que NAO ativar "Correspondencia Avancada Automatica"?
  - Scraping automatico de DOM captura dados sem controle
  - Pode capturar campos sensiveis (senha, tokens)
  - Solucao: Matching manual com dados hasheados (feito no arquivo 04)
